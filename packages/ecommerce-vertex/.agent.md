### Code Style Requirements

**Naming Conventions**:
```typescript
const MAX_RETRIES = 3;              // Constants: UPPER_SNAKE_CASE
const userName = 'John';             // Variables: camelCase
type UserProfile = {};               // Types: PascalCase
interface IDataFetcher {}            // Interfaces: PascalCase (no I prefix)
function calculateTotal() {}         // Functions: camelCase
const user$ = observableUser;       // Observables: $suffix
const count = signal(0);            // Signals: NO suffix
```

**Function Rules**:
- Max 40 lines per function
- Max 5 parameters (use object param for more)
- Explicit return types (NEVER use any)
- Pure functions preferred
- Handle errors explicitly

**File Rules**:
- Max 300 lines per file
- Single responsibility
- Organized imports: external → internal → types

**Comments**:
- Minimal: Only explain WHY, not WHAT
- Self-documenting code (clear names)
- JSDoc for public methods only
- Example:
  ```typescript
  // ❌ BAD: Obvious comment
  const i = 0; // counter
  
  // ✅ GOOD: No comment needed, name is clear
  const retryCount = 0;
  
  // ✅ GOOD: JSDoc for public method
  /**
   * Process payment and create order
   * @param order Order details
   * @returns Promise with transaction ID
   */
  public processPayment(order: Order): Promise<string> { }
  ```

### Async Patterns (Priority Order)

```typescript
// 1️⃣ BEST: Signals for reactive state
readonly count = signal(0);
readonly users = signal<User[]>([]);

// 2️⃣ GOOD: Observables with proper cleanup
constructor(private destroyed$ = inject(DestroyRef)) {}
this.service.data$
  .pipe(takeUntilDestroyed(this.destroyed$))
  .subscribe();

// 3️⃣ ACCEPTABLE: Promises for one-off operations
const data = await service.fetch();

// ❌ AVOID: Manual subscription management
const sub = observable.subscribe();
sub.unsubscribe(); // USE takeUntil instead

// ❌ AVOID: Nested subscriptions
observable.subscribe(() => {
  other$.subscribe(); // NEVER do this
});
```

### Type Safety (Required)

```typescript
// ✅ GOOD: Explicit types everywhere
function getUser(id: string): Promise<User | null> { }
const total: number = items.reduce((sum, item) => sum + item.price, 0);

// ✅ GOOD: Nullish coalescing
const name = user?.name ?? 'Unknown';

// ❌ BAD: No types (auto-inference not allowed)
function getUser(id) { }
const x = something;

// ❌ BAD: Type coercion
const name = user?.name || ''; // Falsy values treated as empty
```

### Test Structure

**Unit Tests** (`*.spec.ts`):
```typescript
// Location: co-located with source (product.service.spec.ts next to product.service.ts)

describe('ProductService', () => {
  let service: ProductService;
  
  beforeEach(() => {
    TestBed.configureTestingModule({ providers: [ProductService] });
    service = TestBed.inject(ProductService);
  });
  
  // Arrange-Act-Assert pattern
  it('should fetch products', async () => {
    // Arrange
    const expected = [{ id: '1', name: 'Product' }];
    
    // Act
    const result = await service.getProducts();
    
    // Assert
    expect(result).toEqual(expected);
  });
  
  // Test success & error cases
  it('should handle API errors', async () => {
    await expectAsync(service.getProducts())
      .toBeRejectedWithError('API Error');
  });
});
```

**E2E Tests** (`cypress/e2e/*.cy.ts`):

## Example: Correct Implementation

**Task**: Add product search feature

✅ **CORRECT**:
```typescript
// domain/product/domain/models/product-search.model.ts
export interface ProductSearchQuery {
  term: string;
  category?: string;
  maxPrice?: number;
}

export interface SearchResult {
  products: Product[];
  total: number;
  took: number; // millis
}

// domain/product/data/services/product-search.service.ts
@Injectable({ providedIn: 'root' })
export class ProductSearchService {
  search(query: ProductSearchQuery): Observable<SearchResult> {
    return this.http.post<SearchResult>('/api/products/search', query)
      .pipe(
        timeout(5000),
        catchError(err => {
          console.error('Search failed', err);
          return throwError(() => new Error('Search unavailable'));
        })
      );
  }
}

// .spec.ts file with tests
describe('ProductSearchService', () => {
  it('should search products', async () => {
    const result = await service.search({ term: 'laptop' }).toPromise();
    expect(result.products.length).toBeGreaterThan(0);
  });
});
```

❌ **WRONG**:
```typescript
// search.ts (wrong file location)
// No DDD structure, no tests
function search(term) {  // no types
  // TODO: implement this later
  // Search for products
  const results = api.search(term); // untyped
  return results;
}
// No test file
```
